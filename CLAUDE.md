## Project

pri-brain - Sistema de organização pessoal baseado no método PARA de Tiago Forte.

## Estrutura PARA

```
1-Projects/   → Projetos com prazo e objetivo definido (ex: lançar site, organizar evento)
2-Areas/      → Responsabilidades contínuas sem prazo (ex: saúde, finanças, carreira)
3-Resources/  → Tópicos de interesse para referência futura (ex: receitas, tutoriais, inspirações)
4-Archive/    → Itens inativos movidos das outras 3 pastas
```

## Regras de Organização

- Projetos concluídos ou abandonados vão para `4-Archive/`
- Áreas nunca terminam, apenas evoluem
- Resources são materiais de apoio, não ações
- Na dúvida entre Projects e Areas: se tem data de término, é Project

## Criando Novos Itens

### Projetos
Criar pasta em `1-Projects/` com `README.md` contendo:
- **Objetivo**: descrição clara do resultado esperado
- **Prazo**: data de conclusão
- **Próximos Passos**: checklist de tarefas

### Áreas e Resources
Criar pasta ou arquivo markdown diretamente na pasta correspondente.
